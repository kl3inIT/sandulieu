<threat_model>

## Trust Boundaries

| Boundary                               | Description                                                                                                    |
| -------------------------------------- | -------------------------------------------------------------------------------------------------------------- |
| route/page inputs → shared contracts   | Untrusted route params and future query values must be normalized before becoming list/detail contract inputs. |
| feature services → shared API mocks    | Feature code depends on the mock API returning backend-shaped DTOs rather than page-owned ad hoc objects.      |
| parent scope → nested resource fetches | Incorrect parent identifiers could return or cache the wrong child dataset.                                    |

## STRIDE Threat Register

| Threat ID | Category | Component                                                  | Disposition | Mitigation Plan                                                                                                                                                          |
| --------- | -------- | ---------------------------------------------------------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| T-01-01   | T        | `shared/api/directory.contracts.ts`                        | mitigate    | Make parent scope and list query types explicit and narrow so later pages/services cannot silently omit required identifiers or substitute display names for stable ids. |
| T-01-02   | I        | `shared/api/department.api.ts`, `shared/api/member.api.ts` | mitigate    | Require explicit scope arguments in raw API signatures so nested data access always carries organization/department context.                                             |
| T-01-03   | D        | paginated list contracts                                   | mitigate    | Use bounded pagination fields (`pageIndex`, `pageSize`) in contract types so later callers implement finite page requests instead of unbounded list fetching.            |
| T-01-04   | R        | mock API boundary                                          | accept      | Mock Phase 1 data has no audit requirement yet; accept lack of mutation audit as deferred until later roadmap phases.                                                    |

</threat_model>
