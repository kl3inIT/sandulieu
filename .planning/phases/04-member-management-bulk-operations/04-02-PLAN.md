Column order per UI-SPEC (left to right):

1. `#` — row index, `text-muted-foreground`, no sort, narrow width
2. Mã thành viên (memberCode) — `font-semibold`, sortable via SortableHeader
3. Họ và tên (fullName) — `font-semibold` primary line + stable ID `member.id` as `text-xs text-muted-foreground` sub-label, sortable
4. Trạng thái (status) — `<DirectoryStatusBadge status={member.status} />`, sortable
5. Thao tác — right-aligned, `text-right`, renders `renderRowActions(member)` or null

Note: The checkbox column is NOT included here — it will be added in Plan 05 (bulk operations). MemberListTable in this plan does NOT include a checkbox column. Plan 05 will extend the table.

Loading state: `<Skeleton>` rows (3 rows, matching column count).
Empty state: Heading "Chưa có thành viên phù hợp" + body "Hãy thay đổi từ khoá tìm kiếm hoặc trạng thái để xem dữ liệu khác trong phạm vi phòng ban hiện tại." — wrapped in a `Card` centered block.
Error state: `<Alert variant="destructive">` with title "Chưa tải được danh sách thành viên" and description from `errorMessage` or default "Đã có lỗi xảy ra khi lấy dữ liệu thành viên."
