**members/new/page.tsx:**

`"use client"` component. Uses `useParams<{ organizationId: string; departmentId: string }>()` and `useRouter()`.

Wire `useCreateMemberMutation(params.organizationId, params.departmentId)`.

On submit:

```typescript
async function handleSubmit(values: MemberFormPayload) {
  await createMutation.mutateAsync(values);
  router.push(
    `/admin/organizations/${params.organizationId}/departments/${params.departmentId}/members`
  );
}
```

On cancel: `router.back()`.

Show inline `<Alert variant="destructive">` if `createMutation.isError`. Title: "Lưu thất bại". Description: `(createMutation.error as Error)?.message || "Không thể lưu thông tin thành viên. Vui lòng kiểm tra lại dữ liệu và thử lại."`.

Generate the `id` for the new member automatically using a stable pattern: `"member-" + crypto.randomUUID().split("-")[0]` — pass as `initialValues.id`. organizationId and departmentId come from route params.

Page layout:

```
<div className="flex flex-col gap-6">
  <h1 className="text-xl font-semibold">Thêm thành viên mới</h1>
  {createMutation.isError && <Alert variant="destructive">...</Alert>}
  <MemberForm
    mode="create"
    initialValues={{ id: generatedId, organizationId: params.organizationId, departmentId: params.departmentId }}
    organizationId={params.organizationId}
    departmentId={params.departmentId}
    isPending={createMutation.isPending}
    onSubmit={handleSubmit}
    onReset={() => router.back()}
  />
</div>
```

Wrap in `<Suspense>` since it uses hooks that need client boundary.

Wire `useMemberDetailQuery(params.organizationId, params.departmentId, params.memberId)` to pre-populate the form.
Wire `useUpdateMemberMutation(params.organizationId, params.departmentId)`.

Loading state: `<Skeleton className="h-64 w-full" />`.
Error state: `<Alert variant="destructive">` with "Chưa tải được thông tin thành viên" title.

On submit:

```typescript
async function handleSubmit(values: MemberFormPayload) {
  await updateMutation.mutateAsync({
    memberId: params.memberId,
    payload: values,
  });
  router.push(
    `/admin/organizations/${params.organizationId}/departments/${params.departmentId}/members/${params.memberId}`
  );
}
```

Redirect after update goes to the member detail page (not the list).

On cancel: `router.back()`.

Show inline error Alert if `updateMutation.isError`. Copy matches create error copy.

Page layout:

```
<div className="flex flex-col gap-6">
  <h1 className="text-xl font-semibold">Chỉnh sửa thành viên</h1>
  {updateMutation.isError && <Alert variant="destructive">...</Alert>}
  {memberDetailQuery.isLoading ? <Skeleton ... /> : (
    <MemberForm
      mode="update"
      initialValues={memberDetailQuery.data}
      organizationId={params.organizationId}
      departmentId={params.departmentId}
      organizationName={memberDetailQuery.data?.parentContext?.organizationName}
      departmentName={memberDetailQuery.data?.parentContext?.departmentName}
      isPending={updateMutation.isPending}
      onSubmit={handleSubmit}
      onReset={() => router.back()}
    />
  )}
</div>
```

Both pages are wrapped in `<Suspense>` with a skeleton fallback.

| ----------------------------------------------- | ------------------------------------------------------------------------- |
| Form submission → useCreateMemberMutation       | Form-validated payload (via memberFormSchema) before reaching mutation    |

## STRIDE Threat Register

| Threat ID | Category  | Component                                   | Disposition | Mitigation Plan                                                                                                                                              |
| --------- | --------- | ------------------------------------------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| T-04-08   | Tampering | Create form — id field                      | mitigate    | id is auto-generated by the page (crypto.randomUUID), not user-entered in the form — schema still validates it. User cannot inject arbitrary IDs via the UI. |
| T-04-09   | Tampering | Create/Edit — organizationId + departmentId | mitigate    | Rendered read-only per D-13. Form schema validates both fields as non-empty, no-whitespace stable IDs. Values sourced from route params, not user input.     |
| T-04-10   | Tampering | Edit form — memberId param                  | accept      | Mock-only scope. getMemberByIdApi scopes lookup to organizationId + departmentId + memberId — cross-dept access not possible via route params.               |
