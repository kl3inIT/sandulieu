## Spacing Scale

Declared values (multiples of 4 only). All tokens map to Tailwind's default scale — use the class names in the Usage column directly.

| Token | Value | Tailwind class | Usage                                             |
| ----- | ----- | -------------- | ------------------------------------------------- |
| xs    | 4px   | gap-1 / p-1    | Icon gaps, checkbox-to-label gap                  |
| sm    | 8px   | gap-2 / p-2    | Compact element spacing, badge gap in table cells |
| md    | 16px  | gap-4 / p-4    | Default element spacing, Card padding             |
| lg    | 24px  | gap-6 / p-6    | Section padding, CardContent grid gap             |
| xl    | 32px  | gap-8 / p-8    | Layout gaps between major blocks                  |
| 2xl   | 48px  | gap-12         | Major section breaks                              |
| 3xl   | 64px  | gap-16         | Page-level top/bottom spacing                     |

Exceptions:

- Checkbox touch target: minimum 44px height on mobile (use `min-h-[44px]` on the row or cell wrapper).
- Bulk action bar height: 56px fixed (py-3 px-4 with content at 32px, giving 12px vertical padding each side).
- Pagination button height: 40px (h-10, consistent with existing `DepartmentListTable` pattern).

## Typography

All sizes use Inter. Two weights only.

| Role                         | Size | Weight         | Line Height | Tailwind classes          |
| ---------------------------- | ---- | -------------- | ----------- | ------------------------- |
| Body                         | 14px | 400 (regular)  | 1.5         | `text-sm`                 |
| Label / table header         | 14px | 600 (semibold) | 1.5         | `text-sm font-semibold`   |
| Card title / section heading | 16px | 600 (semibold) | 1.25        | `text-base font-semibold` |
| Page heading                 | 20px | 600 (semibold) | 1.2         | `text-xl font-semibold`   |

Source: Measured from `DepartmentListTable.tsx` — uses `text-sm`, `text-sm font-medium`, `font-medium`, `text-xs text-muted-foreground`. Weight 500 (medium) is collapsed into 600 (semibold) per two-weight maximum. The `text-xs` (12px) secondary line in table cells (stable ID sub-label) is permitted as a sub-body annotation only — it does not constitute a fifth type size.

## Color

All values are CSS custom properties resolved from `app/globals.css`. The project uses OKLCH tokens — hex approximations are noted for human readability only. Implementations must reference CSS variables, not hardcoded hex.

| Role            | CSS variable         | Light approx              | Usage                                                                                                           |
| --------------- | -------------------- | ------------------------- | --------------------------------------------------------------------------------------------------------------- |
| Dominant (60%)  | `--background`       | #f8fafc (near-white cool) | Page background, table row backgrounds                                                                          |
| Secondary (30%) | `--card` / `--muted` | white / #f3f4f8           | Cards (member list card, detail summary card), sidebar, filter bar background                                   |
| Accent (10%)    | `--primary`          | dark navy blue            | Reserved for: primary action buttons only ("Thêm thành viên", "Lưu thành viên", "Lưu thay đổi", "Xác nhận xoá") |
| Muted text      | `--muted-foreground` | medium gray               | Stable ID sub-labels, pagination description text, empty-state body copy                                        |
| Destructive     | `--destructive`      | red-orange                | Delete button variant, delete dialog confirm button only                                                        |

Accent (`--primary`) is reserved for: primary action buttons only (create CTA, save on form, confirm-delete on dialog). It must NOT be applied to sort headers, filter dropdowns, status pickers, pagination, row actions, or bulk toolbar status picker.

Status badge colors follow `DirectoryStatusBadge` component conventions — do not override.

Bulk action bar background: `--muted` (secondary surface) with a `border-t` separator — same card-like treatment used for the pagination footer in `DepartmentListTable`.

## Interaction Contracts

### Member List Table (MemberListTable)

Column order (left to right):

1. Checkbox (bulk selection) — 40px fixed width
2. # (row index) — `text-muted-foreground`, narrow
3. Mã thành viên (memberCode) — `font-semibold`, sortable
4. Họ và tên (fullName) — primary scan field, `font-semibold` + stable ID sub-label in `text-xs text-muted-foreground`, sortable
5. Trạng thái (status) — `DirectoryStatusBadge`, sortable
6. Thao tác (row actions) — right-aligned, `text-right`

Sorting: column headers 3, 4, 5 are sortable via `SortableHeader` pattern (ArrowUp / ArrowDown / ArrowUpDown icons from lucide-react). Matches `DepartmentListTable` exactly.

Selection state: selecting any row activates `MemberBulkActionBar`. Deselecting all rows hides it. "Select all on page" checkbox in column header selects/deselects all visible rows.

### Bulk Action Bar (MemberBulkActionBar)

Position: inline above the table, inside the `CardContent`, between `MemberListFilters` and the `Table`. Appears when at least 1 row is selected; hidden when 0 rows selected.

Layout: single horizontal row — `flex items-center justify-between gap-4 rounded-md border bg-muted px-4 py-3`.

Left side: "{N} thành viên được chọn" in `text-sm font-semibold`.

Right side: `Select` (status picker) + "Áp dụng" Button (primary variant) + "Bỏ chọn tất cả" Button (ghost variant).

Status picker options: "Hoạt động", "Không hoạt động", "Đã lưu trữ" (maps to `active`, `inactive`, `archived`).

After execution: render inline `Alert` (default variant, not destructive) directly below the bulk bar showing the summary. Dismiss on next bulk action or on row deselect-all.

### Row Actions (MemberRowActions)

Three ghost icon buttons in a flex row, right-aligned:

1. Xem chi tiết — Eye icon — navigates to `[memberId]/page.tsx`
2. Chỉnh sửa — Pencil icon — navigates to `[memberId]/edit/page.tsx`
3. Xoá — Trash2 icon — opens `MemberDeleteDialog` (destructive variant trigger)

No dropdown menu — three inline ghost buttons, consistent with `DepartmentRowActions` pattern. Source: CONTEXT.md D-09.

### Delete Confirmation Dialog (MemberDeleteDialog)

Uses `AlertDialog`. Title: "Xoá thành viên". Description: shows member fullName. Cancel button: "Quay lại". Confirm button: destructive variant, "Xoá". Members have no children so no dependency guard summary is shown. Source: CONTEXT.md D-18, D-19.

### Create / Edit Form (MemberForm)

Layout: Card with `CardHeader` + `CardContent`. Fields rendered with `Field` / `FieldLabel` / `FieldContent` from `shared/components/ui/field.tsx`.

Field order:

1. Tổ chức (organizationId) — read-only text display, not an input
2. Phòng ban (departmentId) — read-only text display, not an input
3. Mã thành viên (memberCode) — `Input`, required
4. Họ và tên (fullName) — `Input`, required
5. Trạng thái (status) — `Select`, required

Read-only fields show the resolved name from route context (not raw UUID). Source: CONTEXT.md D-13.

Form submit button label is context-sensitive:

- Create flow: "Lưu thành viên"
- Edit flow: "Lưu thay đổi"

Form cancel button: "Quay lại" (navigates back without saving).

### Member Detail Page

Layout: Two `Card` blocks stacked vertically with `gap-6`:

1. `MemberDetailSummary` — core fields + parent hierarchy context
2. Actions row — "Quay lại danh sách" (outline), "Chỉnh sửa" (default), "Xoá" (destructive) — consistent with `DepartmentDetailSummary` page pattern

`MemberDetailSummary` grid: `grid gap-4 sm:grid-cols-2`. Fields: ID ổn định, Mã thành viên, Họ và tên, Trạng thái (badge), Tổ chức cha, Phòng ban cha. Source: CONTEXT.md D-14, D-15, D-16, D-17.
